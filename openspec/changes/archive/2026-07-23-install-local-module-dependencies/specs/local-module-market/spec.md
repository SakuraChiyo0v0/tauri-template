## ADDED Requirements

### Requirement: 市场必须让用户预览并确认依赖安装计划
本地模块市场 SHALL 在安装或升级目标包前请求 Host SDK V6 依赖计划，并展示目标模块、自动加入的必需依赖、每个模块的版本动作、权限变化、等待状态和阻塞诊断。只有用户明确确认可执行计划后，市场才能提交该 `planId`。

#### Scenario: 展示多模块安装计划
- **WHEN** 用户选择的模块需要仓库中的两个尚未安装的必需依赖
- **THEN** 市场 SHALL 在确认界面分别展示三个模块、依赖顺序、版本、安装动作和权限摘要

#### Scenario: 用户取消安装计划
- **WHEN** 用户查看计划后选择取消
- **THEN** 市场 MUST NOT 执行 `planId`，当前仓库列表和已安装模块保持不变

#### Scenario: 计划存在阻塞诊断
- **WHEN** 预览结果包含缺包、版本冲突、循环、内容冲突或自动降级诊断
- **THEN** 市场 SHALL 解释具体模块与版本原因并禁用确认操作

#### Scenario: 计划执行后存在等待状态
- **WHEN** 批量安装完成但部分模块等待权限或依赖激活
- **THEN** 市场 SHALL 区分“运行中”“等待权限”和“等待依赖”，不得把包已写入等同于模块已运行

### Requirement: 市场必须处理计划过期而不静默重试
市场 MUST 在 Host SDK 返回 `stale_plan` 时刷新仓库与安装状态，并要求用户重新查看新计划。市场 MUST NOT 在用户没有再次确认时自动执行重新生成的计划。

#### Scenario: 确认时计划已经过期
- **WHEN** 用户确认后基座报告包内容或全局计划已变化
- **THEN** 市场 SHALL 显示计划已变化、重新生成预览，并保持新的确认操作未触发
