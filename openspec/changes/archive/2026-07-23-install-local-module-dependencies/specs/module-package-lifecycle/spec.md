## ADDED Requirements

### Requirement: 多包依赖安装必须作为一个可恢复事务执行
系统 MUST 在单个事务暂存区完整校验和准备安装计划中的全部待写入包，并在全部准备成功后才提升不可变版本目录、原子提交新的全局计划。任何包失败或计划提交前中断时，系统 MUST 撤销本事务新建的版本并恢复最后一个完整计划，不得删除事务前已存在的版本、数据库或授权。

#### Scenario: 批量安装成功
- **WHEN** 已确认计划中的全部包通过最终校验、写入和依赖求解
- **THEN** 系统 SHALL 保存全部新版本并一次提交完整的新计划代次，不暴露只安装部分依赖的中间活动组合

#### Scenario: 一个依赖包最终校验失败
- **WHEN** 计划中的任意包在事务暂存阶段缺失、损坏或不再兼容
- **THEN** 整个事务 SHALL 失败，所有模块版本和当前活动计划保持执行前状态

#### Scenario: 提升版本目录时写入失败
- **WHEN** 部分新版本目录已经由本事务创建但全局计划尚未提交，随后发生文件系统错误
- **THEN** 系统 MUST 删除本事务创建的版本目录并保留执行前计划，不得删除同 ID 的既有版本

#### Scenario: 进程在全局计划提交前中断
- **WHEN** 应用在准备或提升部分版本后退出，事务记录显示新计划代次尚未提交
- **THEN** 下次启动 SHALL 回滚事务创建的版本和暂存内容，并继续使用原计划

#### Scenario: 进程在全局计划提交后中断
- **WHEN** 完整新计划已经原子提交但事务清理尚未完成时应用退出
- **THEN** 下次启动 SHALL 保留已提交的完整计划，只清理事务记录与暂存内容
