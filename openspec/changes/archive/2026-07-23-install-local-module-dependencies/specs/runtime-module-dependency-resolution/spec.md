## ADDED Requirements

### Requirement: 安装预览必须能够考虑授权仓库候选版本
依赖求解器 SHALL 支持在不修改已安装目录的预览模式下，把已完整校验的同一授权仓库包作为候选版本，并区分已安装候选与待安装候选。仓库候选仅用于当前安装计划，不得进入正常启动或普通启停操作的可用版本集合。

#### Scenario: 用仓库候选补齐缺失依赖
- **WHEN** 目标包的必需依赖未安装但授权仓库包含兼容候选
- **THEN** 预览求解 SHALL 选择该候选并把它标记为待安装，而正常运行计划在执行前保持不变

#### Scenario: 仓库候选不能修复整张依赖图
- **WHEN** 候选版本满足目标模块但会使现有期望启用模块无解
- **THEN** 预览 SHALL 返回受影响模块和版本范围，不生成可执行计划

#### Scenario: 仓库候选包含依赖循环
- **WHEN** 目标包与仓库候选组成必需依赖循环
- **THEN** 预览 SHALL 返回稳定的循环路径且不得生成安装顺序

#### Scenario: 普通启动不读取仓库候选
- **WHEN** 应用启动或重新解析已有模块计划但用户没有请求仓库安装预览
- **THEN** 求解器 MUST 只考虑已安装版本，不读取授权仓库或自动安装任何包
