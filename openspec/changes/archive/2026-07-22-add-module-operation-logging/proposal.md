## Why

运行时模块的日志已经能够进入基座日志控制台，但现有模块只在激活和停用时写日志，用户执行新建、刷新、打开等操作后无法从日志判断发生了什么。需要把操作日志变成模块开发的默认规范，并让现有参考模块提供一致、可排查且不泄露敏感数据的日志。

## What Changes

- 模块模板要求使用 Host SDK 日志接口记录生命周期、关键成功操作和可恢复或失败结果。
- 明确日志级别和敏感信息边界：正常操作使用 `info`，可恢复异常使用 `warn`，失败使用 `error`；不得记录正文、凭据或可能携带令牌的完整地址。
- 模块模板示例展示可直接复用的日志写法，并指导后续 AI 在新增操作时同步考虑日志。
- 本地笔记、数据面板和快捷启动三个参考模块补齐用户操作日志，日志来源保持为各自模块 ID。

## Capabilities

### New Capabilities

- `runtime-module-operation-logging`: 规定运行时模块如何向基座日志控制台报告生命周期、关键操作与失败，同时保护敏感数据。

### Modified Capabilities

无。

## Impact

- 模块模板仓库中的开发规范、示例模块和测试。
- 三个本地参考模块的交互处理与测试，以及重新生成的 `.mtp` 包。
- 基座日志接口和日志控制台无需修改；现有 Host SDK ABI 保持兼容。
