## Context

底座当前把中文字符串直接写在 React 页面、主题注册表和模块清单中；源码模块与运行时模块贡献也只接受单个字符串。语言状态不存在，模块只能自行决定文案，无法跟随底座切换。项目仍为 `0.x` 早期阶段，用户明确允许破坏旧协议，因此本次不增加兼容解析分支。

仓库已经归档 `runtime-module-native-capabilities-v1` 变更，Host SDK V3 原生能力与对应主规格已经存在。本次建立 schema V2 和带 i18n 的 SDK V2 基线，并让现有 V3 严格继承该双语能力，不重新设计原生能力边界。

## Goals / Non-Goals

**Goals:**

- 底座只支持 `zh-CN` 与 `en`，默认中文，并在设置中持久化语言选择。
- 切换语言后，不重启即可更新所有底座页面、内置模块贡献和运行时模块宿主贡献。
- 让源码模块和新版 `.mtp` 清单在类型检查或安装校验时必须同时提供中文与英文。
- 给运行时模块页面提供稳定的语言读取和订阅接口。
- 删除 schema V1 与 Host SDK V1 分支，以单一新基线降低后续维护成本。

**Non-Goals:**

- 不支持运行时下载语言包、第三种语言、复数规则、日期货币格式化或自动机器翻译。
- 不翻译开发日志、Rust 原始错误细节、文件名、模块 ID 和数据库内容。
- 不在本变更中新增 SDK V3 原生能力或在线模块更新。
- 不自动删除磁盘上旧协议模块；旧包只是不再能通过新协议校验和激活。

## Decisions

### 使用固定双语值作为公共文案类型

公共契约采用 `LocalizedText = { "zh-CN": string; en: string }`。源码模块名称、说明、导航、设置、选项和主题元数据都使用该类型；运行时 manifest schema V2 使用同一 JSON 形状。固定键比任意 locale map 更容易在 TypeScript 和 Rust 两端证明两种语言齐全，也符合当前只支持两种语言的产品边界。

备选的翻译 key 加全局词典不适合模块，因为底座不应知道第三方模块的 key；任意 locale map 又会让缺少必需语言只能在运行时发现。

### 底座词典与模块词典分离

底座自身使用类型化消息词典和 `t(key, params)` 处理外壳、页面、状态与提示文案。模块宿主贡献直接从清单 `LocalizedText` 解析；模块页面正文由模块自己的词典渲染。这样底座词典不会积累模块业务文案，模块也不能覆盖底座消息。

### 语言状态是独立底座能力

新增轻量语言 store，使用独立 localStorage key 保存 `zh-CN` 或 `en`，默认 `zh-CN`。store 负责订阅、React hook 与 `<html lang>` 同步；设置页语言下拉框直接读写该 store，避免借用模块设置命名空间形成循环依赖。

不引入第三方 i18n 库。当前只有两种语言、无远程词典和复杂格式化，自有类型化实现更小且不会给运行时模块引入额外依赖。

### schema V2 直接替换 schema V1

运行时清单 `schemaVersion` 升级为 2，`name`、`description`、导航标题/说明、设置标签/说明和选项标签全部改为 `LocalizedText`。前后端解析器只接受 schema V2，缺失、空白或额外不受支持的语言都会失败。旧 schema V1 包必须使用新模板重新构建。

Host SDK V1 类型和分发分支删除。SDK V2 保留日志、设置、主题和数据库，并新增 `i18n.getLocale()` 与 `i18n.subscribe(listener)`；现有 SDK V3 保留受控原生能力，并继承 V2 的全部语言与数据库能力。

### 语言切换不重建模块注册表

注册表保存双语原始值，React 页面订阅语言 store，并在渲染时解析当前文案。运行时模块页面通过 Host SDK 的 i18n 订阅自行重绘。语言切换不改变模块激活计划、设置值或页面 ID，因此不触发 WebView 重载或模块重新激活。

## Risks / Trade-offs

- [旧 `.mtp` 包立即失效] → 在安装错误和文档中明确要求用 schema V2 模板重新构建；不自动删除旧模块数据。
- [漏掉硬编码用户文案] → 增加源代码扫描测试并人工切换中英文冒烟，允许开发日志和底层错误保留原语言。
- [已归档 V3 工件保留旧兼容背景] → 不回写历史归档；通过本次 delta spec 更新主规格，使当前行为明确为 schema V2、SDK V2/V3，并拒绝 SDK V1。
- [模块页面内容无法由底座静态检查] → 模板测试覆盖两种语言渲染，并将 Host SDK 语言订阅与模块双语页面列为新模块验收要求。
