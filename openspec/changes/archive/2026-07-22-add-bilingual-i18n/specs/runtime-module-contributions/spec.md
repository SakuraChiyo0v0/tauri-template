## MODIFIED Requirements

### Requirement: 运行时贡献必须由清单驱动
运行时模块 SHALL 通过 schema V2 清单声明导航和基础设置贡献，并 MUST 为模块名称、说明、导航标题/说明、设置标签/说明及选择项标签提供非空的 `zh-CN` 与 `en` 文案。底座 MUST 使用共享外壳按当前语言渲染导航、页面标题、设置控件和主题令牌，不允许模块直接修改共享设置页面或侧边栏代码。

#### Scenario: 注册导航页面
- **WHEN** 已启用模块清单包含合法、不冲突且中英文完整的导航贡献
- **THEN** 对应入口出现在清单指定的导航分组和顺序，并随底座语言切换显示对应标题与说明

#### Scenario: 注册基础设置
- **WHEN** 已启用模块清单包含合法且中英文完整的 `switch` 或 `select` 设置
- **THEN** 设置页面按模块 ID 命名空间显示并持久化设置值，标签、说明和选项随底座语言切换

#### Scenario: 缺少必需语言
- **WHEN** 运行时模块任一宿主渲染文案缺少 `zh-CN`、`en` 或包含空白值
- **THEN** 系统拒绝安装该模块，现有模块和激活计划保持不变

#### Scenario: 贡献标识冲突
- **WHEN** 运行时模块的模块 ID、导航 ID 或自定义元素标签与已注册贡献冲突
- **THEN** 系统拒绝激活该运行时模块，显示冲突错误，并保持其他模块可用

## REMOVED Requirements

### Requirement: 宿主 SDK 必须提供最小稳定能力
**Reason**: 原需求包含继续支持 SDK V1 的旧兼容行为，无法表达当前只接受 schema V2 与 SDK V2/V3 的破坏性新基线。

**Migration**: 使用下方“宿主 SDK 必须提供双语最小稳定能力”需求；SDK V1 模块需要用当前模板重新构建。

## ADDED Requirements

### Requirement: 宿主 SDK 必须提供双语最小稳定能力
底座 SHALL 只向 schema V2 模块提供 SDK V2 或 SDK V3。SDK V2 MUST 提供模块身份、底座版本、分级日志、本模块设置、主题订阅、当前语言读取与订阅以及模块隔离数据库；V3 SDK MUST 继承全部 V2 能力并增加经过声明、用户批准和活动会话校验的文件、进程、注册表、托盘和全局快捷键接口。任何版本均不得暴露内部注册表、任意文件系统路径、原始 Tauri invoke、任意 Shell 或动态原生代码加载。

#### Scenario: V2 模块获得双语与数据库能力
- **WHEN** 已安装 schema V2 模块声明 `sdkVersion: 2`
- **THEN** 底座传入字面量版本为 2、包含语言和隔离数据库接口且不包含 V3 原生能力成员的 SDK

#### Scenario: V3 模块继承语言能力
- **WHEN** schema V2 模块声明 `sdkVersion: 3`、权限已批准且版本进入激活计划
- **THEN** 底座传入包含 V2 语言与数据库能力以及受控原生能力的 V3 SDK

#### Scenario: 模块订阅语言变化
- **WHEN** 活动模块订阅 Host SDK i18n 后用户切换底座语言
- **THEN** 模块收到新的 `zh-CN` 或 `en` 值，且无需重新激活即可重绘页面

#### Scenario: 拒绝旧 SDK
- **WHEN** 模块清单声明 `sdkVersion: 1` 或其他不支持版本
- **THEN** 安装校验失败，现有模块版本、权限和激活计划保持不变

#### Scenario: 模块记录日志
- **WHEN** 运行时模块通过宿主 SDK 写入分级日志
- **THEN** 日志使用该模块 ID 作为来源进入统一日志管道和日志控制台

#### Scenario: 模块读写自身设置
- **WHEN** 模块通过 SDK 读写设置
- **THEN** 设置键被限制在该模块 ID 命名空间，并触发现有设置订阅更新

#### Scenario: 主题发生变化
- **WHEN** 用户切换底座主题模式或主题预设
- **THEN** 运行时页面继承最新 CSS 主题变量，已订阅模块收到主题状态变化通知
