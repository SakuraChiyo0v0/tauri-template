## MODIFIED Requirements

### Requirement: 本地模块包必须经过完整校验
系统 MUST 只安装用户主动选择且符合 schema V2 包协议的 `.mtp` 文件。安装服务 MUST 在修改活动版本前校验 ZIP 结构、清单字段、完整中英文文案、模块标识、语义版本、宿主版本范围、SDK 版本、服务声明、原生能力声明、入口文件、路径安全、文件数量和展开大小。schema V1、SDK V1、缺失任一必需语言、无效服务声明或包含无效 V3/V4 原生能力声明的包 MUST 在保存版本前拒绝。

#### Scenario: 安装有效的新模块
- **WHEN** 用户选择结构有效、双语完整、版本兼容且模块 ID 尚未安装的 schema V2 / SDK V2 模块
- **THEN** 系统将其内容写入该模块版本目录，并按依赖状态决定是否选择该版本

#### Scenario: 安装有效 V3 权限模块
- **WHEN** 用户选择结构有效、双语完整且原生能力声明合法的 schema V2 / SDK V3 模块
- **THEN** 系统保存模块版本和规范化权限摘要，但在用户批准前不激活该版本

#### Scenario: 安装有效 V4 服务模块
- **WHEN** 用户选择结构有效、双语完整、服务声明合法且原生能力声明合法的 schema V2 / SDK V4 模块
- **THEN** 系统保存该版本，并按权限与依赖状态决定是否选择和激活

#### Scenario: 拒绝旧协议模块
- **WHEN** 用户选择 schema V1、SDK V1 或单语言模块包
- **THEN** 系统拒绝安装并提示使用当前双语模板重新构建，且不创建版本目录或改变激活计划

#### Scenario: 拒绝不兼容底座的模块
- **WHEN** 模块清单声明的 `hostVersion` 不包含当前底座版本
- **THEN** 系统拒绝安装并显示兼容性错误，且不创建或切换活动版本

#### Scenario: 拒绝越界路径
- **WHEN** 包中包含绝对路径、父目录跳转、符号链接或解压后超出模块目录的条目
- **THEN** 系统拒绝整个包，且模块安装目录之外的文件 MUST 保持不变

#### Scenario: 拒绝危险原生能力声明
- **WHEN** V3 或 V4 清单请求任意 Shell、提权、HKLM 写入、未知能力或无边界系统范围
- **THEN** 系统拒绝整个包，且既有版本、权限批准和激活计划保持不变

#### Scenario: 校验失败不影响旧版本
- **WHEN** 已安装模块的新包缺少入口、双语文案、服务声明无效或完整性检查失败
- **THEN** 原活动版本继续保持活动，且失败包不得出现在可用版本列表中
