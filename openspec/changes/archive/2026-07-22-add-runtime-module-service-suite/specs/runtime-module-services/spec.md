## ADDED Requirements

### Requirement: 模块必须显式声明并注册服务
schema V2 / SDK V4 模块 MUST 在清单 `services.provides` 中声明允许提供的服务 ID，并 SHALL 通过 Host SDK 注册对应方法。底座 MUST 拒绝未声明、重复、格式无效或由非 V4 模块注册的服务。

#### Scenario: 注册已声明服务
- **WHEN** 活动 SDK V4 模块注册清单中声明且尚未占用的服务 ID
- **THEN** 服务在该模块名下可供合法依赖者调用

#### Scenario: 拒绝未声明服务
- **WHEN** 模块尝试注册清单未声明的服务 ID
- **THEN** 注册失败，既有服务保持不变

#### Scenario: 拒绝重复服务
- **WHEN** 同一模块在未释放旧注册时再次注册相同服务 ID
- **THEN** 第二次注册失败且第一次注册继续有效

### Requirement: 服务调用必须受模块依赖约束
Host SDK V4 MUST 只允许消费者调用自身清单中已声明且当前已选择的必需或可选依赖模块。消费者 MUST 指定提供者模块 ID、服务 ID、方法名和输入；底座不得向模块暴露服务注册表或提供者 Host SDK。

#### Scenario: 调用必需依赖服务
- **WHEN** 消费者依赖的提供者已经激活并注册目标服务与方法
- **THEN** 底座调用提供者方法并把结果返回消费者

#### Scenario: 调用当前存在的可选依赖服务
- **WHEN** 可选依赖版本已进入当前激活计划且注册目标服务
- **THEN** 消费者可以查询服务可用并正常调用

#### Scenario: 拒绝未声明依赖调用
- **WHEN** 消费者调用未在自身清单依赖中声明的模块
- **THEN** 调用在执行提供者代码前失败

#### Scenario: 服务或方法不可用
- **WHEN** 合法依赖没有注册目标服务或方法
- **THEN** 调用返回包含提供者、服务和方法上下文的明确错误

### Requirement: 服务数据必须隔离且可复制
服务输入和输出 MUST 仅包含受支持的 JSON 值，并 MUST 在调用边界进行深度复制。底座 MUST 拒绝函数、类实例、循环引用、危险原型键、非有限数字、过深结构或超限数据。

#### Scenario: 传递合法数据
- **WHEN** 消费者传入合法对象且提供者返回合法对象
- **THEN** 双方收到内容相同但引用隔离的数据副本

#### Scenario: 拒绝非法输入
- **WHEN** 输入包含函数、循环引用或其他不受支持值
- **THEN** 提供者方法不会执行且消费者收到数据边界错误

#### Scenario: 拒绝非法输出
- **WHEN** 提供者返回不受支持或超限值
- **THEN** 消费者不会收到该值，并获得服务返回数据错误

### Requirement: 服务必须随模块生命周期清理
底座 MUST 在模块激活失败、停用、卸载、版本切换或 WebView 计划重载时注销该模块的全部服务。释放后的服务不得继续处理新调用。

#### Scenario: 模块停用后注销服务
- **WHEN** 提供者 Host SDK 被释放
- **THEN** 其全部服务立即不可用，其他模块注册的服务保持有效

#### Scenario: 激活失败清理部分注册
- **WHEN** 模块注册服务后在 `activate` 中抛出错误
- **THEN** 底座释放 SDK、注销已注册服务并进入现有激活失败恢复流程
