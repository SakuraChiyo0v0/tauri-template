## ADDED Requirements

### Requirement: 模块托盘菜单必须由清单声明
V3 模块 SHALL 在清单中声明稳定的托盘菜单 ID、标签、类型和顺序。底座 MUST 命名空间化项目并拒绝重复 ID、未知类型或与底座保留项目冲突的声明。

#### Scenario: 聚合多个模块托盘项目
- **WHEN** 多个活动且已授权模块声明合法托盘菜单
- **THEN** 底座在一个应用托盘中按模块分组和声明顺序展示菜单

#### Scenario: 无模块托盘贡献
- **WHEN** 没有活动模块声明托盘菜单
- **THEN** 底座不会为演示目的创建虚构模块菜单

### Requirement: 托盘事件和更新必须限制在所属模块
Host SDK V3 SHALL 允许模块订阅自己声明项目的点击事件，并更新已声明项目的标签、启用状态和选中状态。模块 MUST NOT 接收其他模块事件、创建清单外项目或修改底座保留菜单。

#### Scenario: 点击模块菜单项
- **WHEN** 用户点击活动模块的托盘项目
- **THEN** 事件只发送给拥有该项目的当前模块会话

#### Scenario: 更新未声明项目
- **WHEN** 模块尝试更新其他模块、底座项目或清单外 ID
- **THEN** 底座返回资源归属错误，现有菜单保持不变

### Requirement: 托盘资源必须跟随模块生命周期
模块停用、回滚、卸载、激活失败、权限撤销或会话失效时，底座 MUST 移除该模块托盘分组和事件路由，同时保持其他模块与底座菜单可用。

#### Scenario: 停用一个托盘模块
- **WHEN** 用户停用一个拥有托盘项目的模块
- **THEN** 只有该模块菜单立即消失，其旧事件订阅不再收到事件
