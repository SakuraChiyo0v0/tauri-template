## MODIFIED Requirements

### Requirement: 本地模块包必须经过完整校验
系统 MUST 只安装用户主动选择且符合受支持包协议的 `.mtp` 文件。安装服务 MUST 在修改活动版本前校验 ZIP 结构、清单字段、模块标识、语义版本、宿主版本范围、SDK 版本、原生能力声明、入口文件、路径安全、文件数量和展开大小。V3 原生能力声明无效或包含底座禁止范围时 MUST 在保存版本前拒绝。

#### Scenario: 安装有效的新模块
- **WHEN** 用户选择一个结构有效、版本兼容且模块 ID 尚未安装的无原生权限模块
- **THEN** 系统将其内容写入该模块版本目录，并按依赖与权限状态决定是否选择该版本

#### Scenario: 安装有效 V3 权限模块
- **WHEN** 用户选择结构有效且原生能力声明合法的 V3 模块
- **THEN** 系统保存模块版本和规范化权限摘要，但在用户批准前不激活该版本

#### Scenario: 拒绝不兼容底座的模块
- **WHEN** 模块清单声明的 `hostVersion` 不包含当前底座版本
- **THEN** 系统拒绝安装并显示兼容性错误，且不创建或切换活动版本

#### Scenario: 拒绝越界路径
- **WHEN** 包中包含绝对路径、父目录跳转、符号链接或解压后超出模块目录的条目
- **THEN** 系统拒绝整个包，且模块安装目录之外的文件 MUST 保持不变

#### Scenario: 拒绝危险原生能力声明
- **WHEN** V3 清单请求任意 Shell、提权、HKLM 写入、未知能力或无边界系统范围
- **THEN** 系统拒绝整个包，且既有版本、权限批准和激活计划保持不变

#### Scenario: 校验失败不影响旧版本
- **WHEN** 已安装模块的新包缺少入口、清单无效、权限声明无效或完整性检查失败
- **THEN** 原活动版本继续保持活动，且失败包不得出现在可用版本列表中

### Requirement: 用户必须能够管理模块生命周期
模块管理界面 SHALL 展示每个运行时模块的选择版本、可用版本、期望启用状态、依赖、依赖者、最近错误、数据库占用、原生权限状态和运行资源，并提供与当前依赖图和权限状态相符的批准、撤销、启用、停用、回滚和卸载操作。系统 MUST 在提交操作前分析直接与间接影响，且不得静默批准新权限、级联修改其他模块或遗留模块系统资源。普通卸载 MUST 删除模块代码和安装状态但默认保留模块数据库、权限批准和外部 grant；已卸载模块的保留数据 MUST 继续可见并可由用户显式清理。

#### Scenario: 批准等待权限模块
- **WHEN** 用户查看完整权限摘要并批准等待中的 V3 模块
- **THEN** 系统保存批准记录并重新解析激活计划，只有依赖也满足时才激活模块

#### Scenario: 启用等待中的模块
- **WHEN** 用户启用一个依赖或权限尚未满足的模块
- **THEN** 系统保存期望启用状态但不加载入口，并显示等待依赖或等待权限诊断

#### Scenario: 停用存在依赖者的模块
- **WHEN** 用户尝试停用一个仍被期望启用模块必需依赖的模块
- **THEN** 系统拒绝操作并返回受影响依赖者，现有激活计划和运行资源保持不变

#### Scenario: 显式安全回滚
- **WHEN** 用户回滚到权限已批准、存在上一计划版本且与当前依赖图兼容的版本
- **THEN** 系统释放当前版本运行资源、提交新计划代次并在重新加载后使用回滚版本

#### Scenario: 回滚版本权限未批准
- **WHEN** 目标回滚版本声明的权限没有有效批准记录
- **THEN** 系统拒绝激活该版本并显示需要审查的权限，不复用当前版本会话或权限

#### Scenario: 回滚会破坏依赖者
- **WHEN** 目标旧版本不满足当前依赖者或需要同时改变其他模块
- **THEN** 系统拒绝单模块回滚、列出受影响模块，并保持当前计划

#### Scenario: 撤销活动模块权限
- **WHEN** 用户确认撤销活动模块的原生权限
- **THEN** 系统先安全停用模块并释放会话、托盘、快捷键和进行中操作，再保存撤销状态

#### Scenario: 卸载存在依赖者的模块
- **WHEN** 用户确认卸载一个仍被期望启用或等待模块必需依赖的模块
- **THEN** 系统拒绝卸载、返回依赖者列表，并保留该模块代码、计划、权限和运行状态

#### Scenario: 卸载无依赖者模块
- **WHEN** 用户确认卸载一个没有必需依赖者的运行时模块
- **THEN** 系统释放全部运行资源、从计划和磁盘删除模块代码，并默认保留模块数据库、权限批准和外部 grant

#### Scenario: 显示已卸载模块保留数据
- **WHEN** 模块已经卸载但其数据库仍存在
- **THEN** 模块管理界面显示该模块 ID、数据库占用和需要明确确认的清理操作

#### Scenario: 操作失败可见
- **WHEN** 安装、权限批准、撤销、启用、停用、升级、回滚或卸载操作失败或等待条件
- **THEN** 模块管理界面显示结构化原因，最后一个完整可用计划及未被明确撤销的授权保持可用
