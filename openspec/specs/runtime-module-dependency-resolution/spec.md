# runtime-module-dependency-resolution Specification

## Purpose

定义本地运行时模块依赖声明、兼容版本求解、拓扑激活、失败隔离和结构化诊断的可观察行为，使独立更新的模块能够安全组成一个确定且可恢复的活动组合。

## Requirements

### Requirement: 模块必须能够声明运行时依赖
运行时模块清单 SHALL 允许声明带语义版本范围的必需依赖和可选依赖。系统 MUST 拒绝自依赖、重复依赖、非法模块 ID 和非法版本范围，同时继续接受没有依赖字段的现有 V1 模块。

#### Scenario: 解析必需和可选依赖
- **WHEN** 模块清单包含合法的 `required` 与 `optional` 依赖列表
- **THEN** 系统保留依赖模块 ID、版本范围和依赖类型用于计划解析

#### Scenario: 兼容无依赖旧模块
- **WHEN** schemaVersion 1 模块清单没有 `dependencies` 字段
- **THEN** 系统把它解析为没有必需或可选依赖的模块

#### Scenario: 拒绝非法依赖声明
- **WHEN** 模块声明自身、重复模块 ID、非法模块 ID 或无法解析的版本范围
- **THEN** 安装校验失败，且不得保存该模块版本

### Requirement: 必需依赖必须决定模块是否可激活
系统 MUST 只为所有必需依赖都存在、已选择且版本范围兼容的模块生成活动选择。依赖不满足的模块 SHALL 保持已安装但未激活，并获得结构化诊断。

#### Scenario: 缺少必需依赖
- **WHEN** 期望启用模块需要一个尚未安装的模块
- **THEN** 该模块不进入活动选择，诊断包含缺失模块 ID 和所需版本范围

#### Scenario: 必需依赖版本不兼容
- **WHEN** 依赖模块已安装但没有可选择版本满足所需范围
- **THEN** 依赖者不进入活动选择，诊断列出依赖模块、所需范围和可用版本

#### Scenario: 可选依赖不可用
- **WHEN** 可选依赖缺失或没有兼容版本
- **THEN** 当前模块仍可激活，且不可用的可选依赖不产生阻塞错误

### Requirement: 解析器必须生成确定的兼容版本组合
系统 SHALL 在本地已安装版本中为每个期望启用模块最多选择一个版本，并使用操作目标、当前选择和版本顺序生成确定结果。找到完整兼容组合前 MUST 不修改当前激活计划。

#### Scenario: 协调选择等待中的兼容升级
- **WHEN** B2 已安装但因需要 A2 而等待，随后安装 A2，且 A2 与 B2 形成完整兼容组合
- **THEN** 新计划同时选择 A2 与 B2，不暴露 A2/B1 或 A1/B2 的中间组合

#### Scenario: 新版本无解时保留当前组合
- **WHEN** 新安装版本与当前依赖图无法形成完整兼容组合
- **THEN** 新版本保留为可用版本，当前选择版本组合保持不变

#### Scenario: 相同输入产生相同结果
- **WHEN** 解析器多次接收相同目录、当前计划和操作偏好
- **THEN** 每次返回相同的选择版本、拓扑顺序和诊断顺序

### Requirement: 必需依赖图必须无环并提供拓扑顺序
系统 MUST 检测已选择必需依赖中的循环，并只为无环选择生成 provider 优先的拓扑激活顺序。同层模块 SHALL 按模块 ID 稳定排序。

#### Scenario: 生成 provider 优先顺序
- **WHEN** 模块 C 依赖 B 且 B 依赖 A
- **THEN** 激活顺序中 A 位于 B 之前，B 位于 C 之前

#### Scenario: 检测直接或间接循环
- **WHEN** 必需依赖形成 A→B→A 或更长循环
- **THEN** 循环中的模块不得进入可激活顺序，诊断包含可读的循环路径

#### Scenario: 独立分支保持可用
- **WHEN** 一个依赖子图存在循环或缺失依赖，而另一个子图完整无环
- **THEN** 系统仍为完整子图生成激活顺序

### Requirement: 上游激活失败必须只阻止依赖者
前端 SHALL 按计划拓扑顺序激活模块。某个模块激活失败时，系统 MUST 跳过所有直接或间接依赖它的模块，同时继续激活不依赖该模块的其他模块。

#### Scenario: provider 激活失败
- **WHEN** A 激活失败且 B、C 直接或间接依赖 A
- **THEN** B 与 C 不执行入口，状态标记为上游激活失败，并包含 A 的模块 ID

#### Scenario: 无关模块继续激活
- **WHEN** A 激活失败而 D 不依赖 A
- **THEN** 系统仍尝试激活 D，并保持其贡献可用

### Requirement: 依赖诊断必须可供管理界面解释
依赖解析结果 MUST 为每个未激活模块提供稳定错误代码、关联模块、所需版本范围、可用版本和循环路径中的适用字段，不得要求前端解析自由文本才能决定状态。

#### Scenario: 展示等待依赖状态
- **WHEN** 模块因缺失或不兼容依赖未被选择
- **THEN** 模块管理页面显示“等待依赖”及具体模块和版本要求

#### Scenario: 展示直接依赖与依赖者
- **WHEN** 用户查看一个已安装运行时模块
- **THEN** 模块管理页面可以显示其直接必需/可选依赖以及直接依赖它的模块
