# module-package-lifecycle Specification

## Purpose

定义本地运行时模块包从校验、安装、升级到失败恢复、回滚和卸载的可观察行为与安全边界，并确保损坏、不兼容或恶意构造的包不会破坏现有可用版本及模块目录之外的数据。

## Requirements

### Requirement: 本地模块包必须经过完整校验
系统 MUST 只安装用户主动选择且符合 V1 包协议的 `.mtp` 文件。安装服务 MUST 在修改活动版本前校验 ZIP 结构、清单字段、模块标识、语义版本、宿主版本范围、SDK 版本、入口文件、路径安全、文件数量和展开大小。

#### Scenario: 安装有效的新模块
- **WHEN** 用户选择一个结构有效、版本兼容且模块 ID 尚未安装的 `.mtp` 包
- **THEN** 系统将其内容写入该模块的版本目录，并把该版本记录为活动版本

#### Scenario: 拒绝不兼容底座的模块
- **WHEN** 模块清单声明的 `hostVersion` 不包含当前底座版本
- **THEN** 系统拒绝安装并显示兼容性错误，且不创建或切换活动版本

#### Scenario: 拒绝越界路径
- **WHEN** 包中包含绝对路径、父目录跳转、符号链接或解压后超出模块目录的条目
- **THEN** 系统拒绝整个包，且模块安装目录之外的文件 MUST 保持不变

#### Scenario: 校验失败不影响旧版本
- **WHEN** 已安装模块的新包缺少入口、清单无效或完整性检查失败
- **THEN** 原活动版本继续保持活动，且失败包不得出现在可用版本列表中

### Requirement: 模块升级必须保留可回滚版本
系统 SHALL 在安装更高且兼容的新版本时保留当前活动版本，并在新版本完整落盘后才切换活动版本。

#### Scenario: 升级已安装模块
- **WHEN** 用户安装同一模块 ID 的更高兼容版本
- **THEN** 新版本成为活动版本，原活动版本成为上一版本，两个版本都可在磁盘状态中识别

#### Scenario: 拒绝重复或较低版本覆盖
- **WHEN** 用户通过安装操作提交与活动版本相同或更低的版本
- **THEN** 系统拒绝覆盖，并提示使用回滚操作恢复旧版本

### Requirement: 激活失败必须安全恢复
系统 MUST 接收前端激活结果。新版本激活失败时，有上一版本则恢复上一版本；没有上一版本则阻止失败版本再次自动激活，并保留可诊断错误。

#### Scenario: 升级版本激活失败
- **WHEN** 新活动版本的入口导入、`activate` 调用或页面元素校验失败，并且存在上一版本
- **THEN** 系统恢复上一版本为活动版本、记录失败版本和错误，并允许底座重新加载恢复后的模块

#### Scenario: 首次安装后激活失败
- **WHEN** 首次安装的模块激活失败且不存在上一版本
- **THEN** 系统保留安装记录但将该版本标记为激活受阻，底座继续运行且不反复加载该版本

### Requirement: 用户必须能够管理模块生命周期
模块管理界面 SHALL 展示每个运行时模块的活动版本、上一版本、启用状态和最近错误，并提供与当前状态相符的停用、启用、回滚和卸载操作。

#### Scenario: 显式回滚
- **WHEN** 用户对存在上一版本的模块执行回滚
- **THEN** 上一版本成为活动版本，原活动版本保留为可用版本，并在重新加载后使用恢复版本

#### Scenario: 卸载运行时模块
- **WHEN** 用户确认卸载一个运行时模块
- **THEN** 系统删除该模块的所有已安装代码和活动状态，并在重新加载后移除其页面、设置与启用状态

#### Scenario: 操作失败可见
- **WHEN** 安装、升级、回滚或卸载操作失败
- **THEN** 模块管理界面显示可操作的错误信息，且当前可用模块保持可用

### Requirement: 安装状态必须可恢复
模块版本状态 MUST 使用防止部分写入的方式保存。进程在包复制或状态切换期间中断时，下一次启动 SHALL 忽略未完成暂存内容并恢复最后一个完整状态。

#### Scenario: 安装过程中进程中断
- **WHEN** 应用在暂存包内容完成前退出
- **THEN** 下一次启动继续使用原活动版本，并可安全清理未完成的暂存目录

#### Scenario: 状态写入过程中断
- **WHEN** 应用在切换活动版本状态时意外退出
- **THEN** 下一次启动读取到旧状态或完整新状态之一，不得读取到无法解析的部分状态
