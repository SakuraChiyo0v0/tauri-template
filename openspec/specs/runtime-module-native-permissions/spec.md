## Requirements
### Requirement: V3 模块必须声明原生能力
Host SDK V3 模块 MUST 在清单中声明可能使用的原生能力及其最小静态范围。安装服务 MUST 拒绝未知能力、无效范围、重复贡献以及底座明确禁止的高风险请求，且不得根据模块代码自动扩大声明。

#### Scenario: 校验合法能力清单
- **WHEN** V3 模块只声明底座支持且范围有效的文件、进程、注册表、托盘或快捷键能力
- **THEN** 安装服务保存规范化声明并生成可供用户阅读的权限摘要

#### Scenario: 拒绝越界能力声明
- **WHEN** 模块请求任意 Shell、管理员提权、HKLM 写入、原始 Tauri invoke 或未知能力
- **THEN** 安装在写入版本目录前失败，当前活动计划和已授权权限保持不变

### Requirement: 原生权限必须由用户明确批准
V3 模块的原生能力声明 MUST 在首次激活前由用户完整查看并明确批准。包可以在未批准时保存为可用版本，但 MUST NOT 进入激活计划、创建原生会话或注册系统资源。

#### Scenario: 首次安装等待批准
- **WHEN** 用户安装包含原生能力声明的新模块但尚未批准权限
- **THEN** 模块显示等待权限状态，不执行入口且不产生文件、进程、注册表、托盘或快捷键副作用

#### Scenario: 批准全部声明权限
- **WHEN** 用户查看权限摘要并确认全部声明
- **THEN** 底座保存与模块版本声明指纹绑定的批准记录，并允许依赖解析器选择该版本

#### Scenario: 升级扩大权限
- **WHEN** 新模块版本增加能力或扩大已有范围
- **THEN** 新版本保持等待批准，旧版本和旧权限继续工作，系统不得把旧批准自动套用到扩大后的声明

#### Scenario: 升级不扩大权限
- **WHEN** 新模块版本的规范化权限声明与已批准声明相同或更窄
- **THEN** 底座可以复用现有批准记录并继续正常的版本选择流程

### Requirement: 原生调用必须绑定活动会话
底座 MUST 只为当前激活计划选择、权限已批准且正在激活的 V3 模块版本签发不可预测的会话令牌。每次原生调用 MUST 校验令牌、模块版本、能力声明、用户批准和资源归属。

#### Scenario: 活动模块使用已批准能力
- **WHEN** V3 Host SDK 使用当前有效会话调用清单已声明且用户已批准的能力
- **THEN** Rust 代理执行对应的受控操作并把结果只返回给该会话

#### Scenario: 拒绝伪造或过期会话
- **WHEN** 调用使用未知令牌、旧版本令牌或已停用模块的令牌
- **THEN** 底座返回结构化 `invalid_session`，且不触碰目标系统资源

### Requirement: 权限撤销必须先释放运行资源
用户 SHALL 能够查看并撤销模块原生权限。撤销任何已批准权限时，底座 MUST 先停用模块、注销会话、托盘项和快捷键并终止由会话管理的进行中操作，再保存撤销状态。

#### Scenario: 撤销活动模块权限
- **WHEN** 用户确认撤销一个正在运行模块的原生权限
- **THEN** 模块贡献和原生资源先被清理，随后模块进入等待权限状态且旧会话调用全部失败

#### Scenario: 卸载后保留权限决定
- **WHEN** 用户普通卸载模块但没有选择清除模块数据与授权
- **THEN** 底座删除模块代码并释放运行资源，但保留文件 grant 和批准记录供同 ID 兼容版本重新安装时审查复用
